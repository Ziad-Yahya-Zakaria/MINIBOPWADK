# Free Engine Bulk Spec

هذه الوثيقة تعرف آلية تبادل البيانات الدورية بين المواقع أو الجهات المختلفة عبر حزم ملفات.

## الهدف

- تبادل البيانات بدون ربط مباشر بين قواعد البيانات.
- تمكين رفع ملف دوري إلى موقع مركزي أو مشاركته يدوياً.
- تمكين المواقع الأخرى من استيراد الملف ورؤية البيانات والتقارير محلياً.

## السيناريو

1. المستخدم يختار فترة أو مجموعة تقارير.
2. النظام ينشئ `Bulk Package`.
3. الحزمة يتم رفعها إلى موقع مركزي أو نقلها يدوياً.
4. الجهة الأخرى تنزّل الحزمة.
5. الجهة الأخرى ترفع الحزمة داخل Minibo Systems لديها.
6. النظام يستورد البيانات ويعرضها بحسب الصلاحيات.

## شكل الحزمة

- الامتداد المقترح: `.zip`
- المحتوى:
  - `manifest.json`
  - `data/production-batches.json`
  - `data/production-entries.json`
  - `data/approvals.json`
  - `data/products.json`
  - `reports/*.xlsx` اختياري
  - `checksums.json`

## manifest.json

يجب أن يحتوي على الأقل على:

- `schemaVersion`
- `packageId`
- `sourceInstanceId`
- `sourceInstanceName`
- `periodFrom`
- `periodTo`
- `exportedAtUtc`
- `entityCounts`
- `checksum`

## قواعد الاستيراد

- منع استيراد الحزمة نفسها مرتين
- التحقق من سلامة الحزمة قبل فتحها
- ربط كل سجل بمصدره `sourceInstanceId`
- تعليم البيانات المستوردة بأنها imported أو read-only إذا لزم
- تسجيل كل import وexport في `Audit Log`

## واجهة الإدارة

- إنشاء حزمة جديدة
- تنزيل الحزمة
- رفع الحزمة
- سجل الحزم المصدرة
- سجل الحزم المستوردة
- عرض أخطاء التحقق
- إعادة فحص checksum

## الجدولة

- الادمن يحدد فترة التصدير
- يمكن إنشاء حزم:
  - يومية
  - أسبوعية
  - شهرية
  - يدوية

## الصلاحيات

- `Bulk.Export`
- `Bulk.Import`
- `Bulk.Download`
- `Bulk.Manage`

## ملاحظات مهمة

- البيانات المستوردة يجب أن تكون قابلة للتصفية في التقارير
- يجب أن يمكن استثناء البيانات المستوردة من بعض التقارير
- لا تعتمد على اسم الملف فقط
- افحص الـ zip والـ json والـ checksum قبل القبول
