10) حدود الثقة Security Boundaries
==================================================

 المنطقة الأولى:
 - واجهة React
 - تعتبر طبقة عرض فقط

 المنطقة الثانية:
 - AppContext
 - تعتبر حارس الجلسة والصلاحيات

 المنطقة الثالثة:
 - IndexedDB
 - تمثل مخزن التشغيل المحلي

 المنطقة الرابعة:
 - ملفات التصدير والاستيراد
 - تعتبر حدود تبادل بين النظام المحلي وأنظمة أو بيئات أخرى

 المنطقة الخامسة:
 - نقاط التكامل المستقبلية
 - JD Edwards / Salesforce / Custom APIs


==================================================
11) أهم نقاط القوة الحالية
==================================================

 - Local-first architecture مناسبة للتشغيل السريع وبيئات الانقطاع.
 - إدخال جدولي قوي عبر TanStack Table.
 - ربط مباشر بين الإنتاج الخام والتقرير النهائي.
 - صلاحيات قابلة للتخصيص عبر Profiles و Permissions.
 - جلسات token-based مع انتهاء وخمول وإبطال.
 - تصدير مؤسسي لـ PDF/XLSX.
 - منفذ جاهز للتكاملات المؤسسية.


==================================================
12) المسار المؤسسي التالي
==================================================

 - إضافة API Gateway حقيقي خلف AppContext.
 - نقل sessions و users و approvals إلى Backend مركزي.
 - إضافة Audit Trail لكل عملية.
 - إضافة Queue للتكاملات الصادرة والواردة.
 - إضافة Mapping Engine فعلي لكل مزود خارجي.
 - دعم SSO / SAML / OIDC للحسابات المؤسسية.
 - إضافة Web Worker لتصدير الملفات الكبيرة.
